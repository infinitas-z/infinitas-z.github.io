1-什么是JSP页面

jsp(java server pages): java 服务端页面，一个既可以写java也可以写html的页面

2-request请求对象

2.1相关概念

  • request请求方式：最常用的是GET请求和POST请求
  • GET请求：通常在超链接中使用
  • GET特点：请求的参数在URL后出现且有长度限制，安全性不高
  • POST请求：通常在表单提交中使用，中文需要编码处理
  • POST特点：请求参数在请求体中，没有长度限制，安全性高

2.4 重定向

2.4.1 页面重定向：

void sendRedirect(String location) ， 客户端将重新发送请求到指定URL

重定向是在客户端发挥作用，通过发送一个新的请求来实现页面的转向，在地址栏中可以显示转向后的地址

案例：

2.4.2 请求转发

转发：是在服务端发挥作用的，将同意请求在服务器资源之间进行传递，客户端浏览器的地址栏不会显示转向后的地址

5- Cookie

5.1 概念方法

概念：Cookie是Web服务器保存在客户端的一系列文本信息，是客户端会话技术

作用：对特定对象的追踪，实现各种个性化服务，简化登录

缺陷：安全性不高，容易泄露信息

6-Servlet

6.1 概念

Servlet( Server Applet) 运行在服务端的小程序，简单理解就是一个接口，定义了Java类被浏览器访问到的规则

Servlet执行原理

  1. 当服务器接收到客户端浏览器请求后，会解析请求URL路径，获取访问的Servlet的资源路径
  2. 查找web.xml文件，是否有对应的标签体内容
  3. 如果有，则在找到对应的全类名
  4. tomcat会将字节码文件加载进内存，并且创建对象
  5. 调用其方法
